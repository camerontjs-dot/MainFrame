# New Project Workflow

A new-project workflow should create the minimum viable project scaffold. This ensures every project starts from a standard operating shape rather than becoming a one-off folder with inconsistent artifacts.

## Required Scaffold:
When starting a new project in `30_projects/`, create the following:
- `README.md` or `BRIEF.md`: Goal, status, and next steps.
- `log.md`: Execution trace and changes.
- `decisions.md`: Local decisions that don't yet warrant global ADRs.
- Subdirectories for `plans/`, `raw-materials/`, and `outputs/`.
